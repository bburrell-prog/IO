╔══════════════════════════════════════════════════════════════════════════════╗
║                    DESKTOP SCREEN ANALYZER - QUICK START                     ║
╚══════════════════════════════════════════════════════════════════════════════╝

STEP 1: Navigate to the project directory
═══════════════════════════════════════════════════════════════════════════════
cd "/Users/bburrell/Desktop/IO/py"


STEP 2: Run the automated setup script
═══════════════════════════════════════════════════════════════════════════════
./setup.sh

This will:
- Check for Python 3 and Tesseract
- Create a virtual environment
- Install all dependencies
- Create a .env configuration file


STEP 3: Add your OpenAI API key
═══════════════════════════════════════════════════════════════════════════════
nano .env

Replace: sk-your-api-key-here
With your actual OpenAI API key from: https://platform.openai.com/api-keys

Save with: Ctrl+O, Enter, Ctrl+X


STEP 4: Run the application
═══════════════════════════════════════════════════════════════════════════════

OPTION A - Single Analysis (Recommended for macOS - No sudo required):
───────────────────────────────────────────────────────────────────────────────
source venv/bin/activate
python3 main.py --once

This captures one screenshot, analyzes it with ChatGPT, and exits.


OPTION B - Interactive Mode (Requires sudo on macOS):
───────────────────────────────────────────────────────────────────────────────
sudo ./venv/bin/python3 main.py

- Press F9 to trigger screen analysis
- Press ESC to exit


OPTION C - Use the quick-start script:
───────────────────────────────────────────────────────────────────────────────
./run.sh

This interactive script will guide you through running the application.


═══════════════════════════════════════════════════════════════════════════════
                              TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "Tesseract not found"
Solution: Install with Homebrew:
  brew install tesseract

Problem: "keyboard library requires root access"
Solution: This is normal on macOS. Either:
  - Use --once mode (no sudo needed)
  - Run with sudo as shown in Option B

Problem: "OPENAI_API_KEY not found"
Solution: Make sure you edited .env with your actual API key

Problem: Module not found errors
Solution: Make sure virtual environment is activated:
  source venv/bin/activate
  pip install -r requirements.txt
